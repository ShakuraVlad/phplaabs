# Лабораторная работа №7 — Шаблонизация PHP

## Описание проекта

**Книга рецептов** — веб-приложение для добавления, просмотра и удаления кулинарных рецептов.  
Данные хранятся в `data/recipes.json`. Реализованы два режима отображения:

| Режим | URL | Шаблонизатор |
|---|---|---|
| Нативный PHP | `index.php?mode=native` | output buffering + `require` |
| Twig | `index.php?mode=twig` | Twig 3, наследование шаблонов |


## Дополнительное задание (10 баллов)

Зарегистрирован кастомный Twig-фильтр **`format_duration`**:

```twig
{{ recipe.prep_time | format_duration }}
{# 90  → "1 ч 30 мин" #}
{# 10  → "10 мин"      #}
{# 120 → "2 ч"         #}
```

Регистрация в `index.php`:
```php
$twig->addFilter(new \Twig\TwigFilter('format_duration', function (int $minutes): string {
    return formatDuration($minutes);
}));
```

## Контрольные вопросы

### 1. Нативные PHP-шаблоны vs Twig

| Критерий | Нативный PHP | Twig |
|---|---|---|
| Синтаксис | Полный PHP (любая логика) | Ограниченный DSL |
| Безопасность | XSS вручную (`htmlspecialchars`) | Автоэкранирование по умолчанию |
| Наследование | Только через `include`/буферизацию | `extends` + `block` |
| Скорость разработки | Ниже (больше boilerplate) | Выше (лаконичный синтаксис) |
| Производительность | Чуть быстрее (нет парсинга) | Кеширование компенсирует |
| Порог вхождения | Нулевой для PHP-разработчика | Нужно изучить синтаксис Twig |

### 2. Зачем разделять логику и представление?

Смешивание ведёт к:
- **Дублированию кода** — одна правка нужна в нескольких местах.
- **Сложности тестирования** — нельзя протестировать логику отдельно от HTML.
- **Трудному сопровождению** — верстальщик не должен разбираться в SQL/бизнес-логике.

Разделение даёт: читаемость, повторное использование шаблонов, параллельную работу команды.

### 3. Наследование шаблонов в Twig

`layout.html.twig` определяет скелет страницы с именованными блоками:
```twig
{% block content %}{% endblock %}
```

`index.html.twig` расширяет его и заполняет блоки:
```twig
{% extends "layout.html.twig" %}
{% block content %}
    <!-- конкретный HTML этой страницы -->
{% endblock %}
```

Twig подставляет содержимое блоков в нужные места макета. Это позволяет иметь один макет и множество страниц без копипасты header/footer.
